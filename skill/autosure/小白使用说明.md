# Autosure 小白使用说明（skymini）

> **当前版本提示（2026-04-19 closeout）**
>
> - 产品默认路线仍是 **userscript**。
> - 在 mini49 当前机器上，优先可用路径仍是 **native fallback**。
> - `/autosure N` 现在不是“先排队再等很久”，而是：**首轮立即前台续跑，剩余轮次按每轮完成后继续**。

## 新电脑 / 给客户：一键思路（先看这个）

1. 把整个 **`autosure`** 文件夹放到 OpenClaw 的 workspace：`.../skills/autosure`。  
2. 终端执行：  
   `python3 skills/autosure/scripts/install_bundle.py --mode full`  
3. 把脚本打印出来的 **`plugins` JSON** 合并进 `~/.openclaw/openclaw.json`（和你机器上原有插件配置**手工对齐**，不要弄丢别的插件）。  
4. `openclaw config validate` → `openclaw plugins inspect autosure-resume` → **重启网关**。  

更细的给「部署 Agent」看的：**`HANDOFF_FOR_AGENT.md`**；英文与结构说明：**`DEPLOY.md`**；架构与需求：**`docs/`**；发版故事：**`RELEASE_NOTES.md`**。

## 你会得到什么

`autosure` 是给 OpenClaw 无人值守任务加“保险”的一套机制：

- 任务中断后可自动续跑（不是无限循环）
- 连续失败会自动熔断，避免会话风暴
- 有去重和等待窗口，避免重复发送
- 有收口模板，方便你查看当前状态

## 现在的默认安全值

- 最多自动续跑：`1` 次
- 续跑冷却：`15` 秒
- 上下文压缩等待：`60` 秒
- 熔断阈值：连续失败 `3` 次后打开

## 日常怎么用（最简单）

1. 正常在 OpenClaw 里发任务
2. 想让它自动连跑：发 **`/autosure`**（无数字 = **不限轮次**，直到你发 **`/autosure stop`** 或发一条普通消息清掉循环）；或发 **`/autosure 数字`**（只替你代发这么多次；数字上限可在插件配置 `commandMaxRounds` 里调大，也支持紧写如 `/autosure10`）
3. **自动「继续」**：每轮助手答完停住后，默认会等 **约 15 秒**；这段时间里你**什么都不发**，系统才替你发一条 `发动词.txt` 里的短句（轮换）；若你**发了任意一条消息**，就用你的内容继续，**不会**再替你发这一轮。
4. 如果任务中断，autosure 会在安全规则内自动续跑（与上面「成功连轮」是另一条逻辑）
5. 你有新想法，直接输入新消息，人工优先（会覆盖自动循环）

## 自定义发动词（轮换）

编辑本目录下的 `发动词.txt`：一行一条短句，插件会按行轮换，当作你在 Web 里发出的「继续」类用户消息（无需改插件代码）。

## 口令说明（当前会话生效）

- `/autosure-demo`：只在**当前 OpenClaw 对话**里做一轮可视化（先看说明，再自动注入发动词并触发模型短答；约 8 秒内不要重复发）
- `/autosure`：**不限轮次**代发发动词（规则同上；费用与是否停由你自己把握）
- `/autosure 10`：在你本条指令之外，**替你代发 10 次**发动词（每轮助手停住 → 等窗口 → 无人输入才代发）
- `/autosure status`：查看当前轮次状态
- `/autosure stop`：立刻停止自动轮跑

## 什么时候需要你手动处理

- 连续失败导致熔断（系统会停止自动续跑）
- 上游模型长期不可用
- 你要跨越高风险 gate（例如 D/F）

## 一键脚本

桌面脚本：`Autosure-启动与巡检.command`

它会自动：
- 检查网关状态
- 检查 autosure 插件状态
- 跑一次健康检查脚本
- 打开 OpenClaw dashboard
