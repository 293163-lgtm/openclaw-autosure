# Autosure for OpenClaw — PRD（v2.1）

## 1. 北极星体验

用户离开几分钟后回来，看到任务在自动推进、没有风暴、没有跑偏、自己随时插话即可接管。

一句话：**像一个靠谱副手在替你继续，但永远听你的。**

## 2. 版本定位

- **v2.0**：可分发 skill + vendored 插件 + 一键安装。
- **v2.1（本轮）**：在 v2.0 基础上加入「可视化胶囊（A1 userscript）」与「取消双保险」等体验升级；且保证胶囊失效时核心能力不受影响。

> 与旧版不冲突原则：旧命令与旧配置继续可用；新增能力默认可选、可关闭。

## 3. 产品目标

1. **更直观**：用户不用猜系统状态（运行/等待/停止）。
2. **更可控**：用户可通过胶囊快捷启动/停止与 3/6/9 轮。
3. **更稳健**：升级 OpenClaw 后，即使胶囊失效，autosure 核心继续工作。
4. **更可修复**：提供「胶囊修复包」与开关脚本。

## 4. 用户故事

1. 作为用户，我要在 chat 页看到胶囊状态（空闲/运行中/暂停）。
2. 作为用户，我要一键触发 `/autosure 3|6|9`，不想每次打命令。
3. 作为用户，我点「暂停」时，语义必须明确等价于 **`/autosure stop`**（B1）。
4. 作为运维，我升级 OpenClaw 后即使胶囊挂了，也不影响插件核心行为。
5. 作为运维，我可以用修复包更新胶囊注入逻辑并一键启停。

## 5. 功能范围（MoSCoW）

### Must

- 胶囊实现方式：**A1 userscript**（旁路注入，不改 OpenClaw 本体）。
- 胶囊位置策略：优先挂载在 chat 区「刷新」附近；找不到则降级为安全位置（右上浮层）。
- 胶囊动作：
  - 启动快捷：`/autosure 3`、`/autosure 6`、`/autosure 9`
  - 启动不限轮：`/autosure`
  - 暂停：**`/autosure stop`**
  - 状态刷新：`/autosure status`
- 修复包：
  - 更新注入脚本
  - 启用/禁用胶囊
  - 提供回滚入口
- 核心插件增强：取消双保险（除 `before_dispatch` 外，再补 inbound 路径，降低漏取消概率）。

### Should

- 胶囊本地状态持久化（最后选择轮次、上次状态）。
- 关键事件日志（胶囊注入成功/失败、命令下发结果）。

### Could

- 胶囊主题跟随（暗黑/亮色）。
- 胶囊微动效与折叠展开动画。

### Won’t（本轮不做）

- 不改 OpenClaw 主仓 UI 代码。
- 不承诺通过胶囊直接读取所有内部私有状态 API。

## 6. 非功能要求

| 维度 | 要求 |
|------|------|
| 稳定性 | 胶囊不可用不影响 `autosure-resume` 主流程 |
| 兼容性 | OpenClaw 2026.4.x；DOM 变化时可通过修复包适配 |
| 可回退 | 一键禁用胶囊后，命令行与手工输入完全可用 |
| 可维护 | 修复包独立版本号；与 skill 主版本并行追踪 |

## 7. 验收标准

1. 胶囊可在 chat 页面显示并执行 3/6/9 快捷命令。
2. 点击暂停，实际发送并生效 ` /autosure stop`。
3. 升级 OpenClaw 后（模拟 DOM 改动），禁用胶囊时 autosure 核心能力不受影响。
4. 修复包可恢复胶囊可见与可操作。
5. `verify_health.py` 与 `plugins inspect autosure-resume` 仍通过。

## 8. 风险与缓解

- DOM 选择器失效：通过多级 selector + 浮层降级 + 修复包更新。
- 通道差异导致取消漏判：引入 inbound 路径映射，保留日志可追。
- 用户误解暂停语义：UI 文案固定为“暂停（发送 /autosure stop）”。
