# Autosure UI 胶囊专项设计（A1 / B1）

## 1. 决策冻结

- **A1（默认路线）**：胶囊实现使用 userscript（Tampermonkey 兼容），不改 OpenClaw 本体。
- **A1-Fallback（运维修复路线）**：当 Tampermonkey 已安装脚本但本机 `127.0.0.1:18789/chat*` 不执行时，可使用 native fallback，把同一套胶囊注入本机 OpenClaw Control UI。
- **B1**：暂停按钮语义固定为 `/autosure stop`。
- **UI vNext 冻结方向**：
  - 默认 **收起态**
  - 锚定 **OpenClaw 顶部刷新图标左边**
  - 收起态为 **图标 + 状态点**
  - 展开方向为 **向下**
  - 按钮布局为 **两排**
  - 视觉气质 **跟随 OpenClaw 原生图标与工具栏风格**

## 2. 胶囊目标

1. 默认几乎不打扰聊天正文，不遮挡核心阅读区域。
2. 让 autosure 在视觉上像 OpenClaw 的**原生控制工具**，而不是外挂浮层。
3. 一键执行常见轮次（3/6/9/不限）、停止与状态查询。
4. 升级后可通过修复包快速恢复挂载。

## 3. 交互设计

### 3.1 总体形态

胶囊采用 **两态设计**：

- **收起态**：只显示一个小触点（图标 + 状态点）
- **展开态**：显示完整 autosure 控制面板

默认进入页面时为 **收起态**。

### 3.2 收起态（Closed State）

#### 位置

- 首选：**刷新图标左边**
- 必须位于 OpenClaw 顶部工具区内，与现有 toolbar 保持同层级

#### 组成

- Autosure 图标
- 状态点（无文字）

#### 状态点语义

- **灰**：idle / 已停止
- **绿**：running / 自动轮跑中
- **黄**：pending / 状态同步中、待确认
- **红**：error / 熔断、发送器异常、显式故障

#### 原则

- 不显示长文字
- 不占正文区域
- 不采用右下角大浮层作为默认展示

### 3.3 展开态（Expanded State）

#### 展开方向

- **向下展开**

#### 面板结构

1. **头部**
   - Autosure 标题
   - 当前状态文案
   - 收起/关闭操作

2. **按钮区（两排）**
   - 第一排：`3` `6` `9` `∞`
   - 第二排：`Stop` `Status`

3. **底部状态区**
   - 一行轻量状态反馈
   - 示例：
     - 运行中
     - 已停止
     - 状态同步中
     - 等待注入
     - 剩余 N 轮

### 3.4 打开 / 关闭逻辑

#### 打开

- 点击收起态图标 → 展开

#### 关闭

- 再点一次图标 → 收起
- 点击胶囊外部区域 → 收起
- 页面切换/重进后默认回到收起态

#### 明确不做

- hover 展开
- 默认常驻展开
- 自动大面积占位

## 4. 视觉设计

### 4.1 风格原则

胶囊必须**跟随 OpenClaw 工具栏的系统风格**：

- 图标语言一致
- 圆角体系一致
- 阴影与层级一致
- hover / active / focus 节奏一致
- 看起来像 OpenClaw 内建功能，而不是第三方外挂

### 4.2 不推荐样式

- 右下角常驻大黑块
- 明显“浏览器插件感”的悬浮窗
- 过重、过亮、过科幻的独立视觉体系
- 长标签、小作文式说明文字

### 4.3 推荐样式

- 收起态：小而克制，靠图标与状态点表达存在感
- 展开态：精致胶囊 / 下拉工具舱
- `Stop` 按钮可比普通按钮更醒目，但不做惊悚红色大按钮
- `Status` 弱于 `Stop`，体现“查询”而非“控制”

## 5. 技术设计

### 5.1 组件与注入分层

胶囊实现应拆为两层：

1. **注入层（Mount Strategy）**
   - 负责寻找顶部锚点
   - 负责把胶囊插到刷新图标左边
   - 找不到锚点时负责降级挂载

2. **组件层（Capsule Component）**
   - 负责收起态 / 展开态 UI
   - 状态点
   - 两排按钮
   - 点击外部关闭
   - 状态反馈与微动画

> userscript 与 native fallback 必须复用同一套组件结构与交互规格，区别只在“注入路径”，不能演化成两套产品。

### 5.2 注入入口

- `@match` 规则覆盖 OpenClaw control-ui 地址
- `MutationObserver` 监听页面结构变化，实现热挂载
- native fallback 注入本机 Control UI `index.html` 时，也必须遵守同一规格

### 5.3 命令发送器

- 查找 chat 输入框 `textarea`
- 写入命令后触发 `input` 事件
- 触发发送按钮 click（`aria-label=Send message / Queue message` 等多候选）
- 首次挂载后自动发送一次 `/autosure status` 同步状态
- 快捷动作后自动追加一次 `/autosure status` 进行状态校准

### 5.4 挂载选择器策略

#### 正式优先级

1. 刷新图标按钮（语义或文案命中 Refresh / 刷新）
2. 顶部 chat 工具栏中刷新按钮邻近容器
3. 顶部工具区备用锚点
4. 右上角**收起态小图标**降级

#### 明确废弃为默认方案

- 右下角常驻大浮层

右下角大浮层只允许作为最后兜底的应急形态，且即便兜底，也应优先保持**收起态**而不是直接大面板常驻。

## 6. 修复包设计

建议目录：`ui-capsule/repair-pack/`

- `autosure-capsule.user.js`：主 userscript（版本号）
- `native-control-ui-capsule.js`：native fallback 组件源码
- `selectors.json`：选择器与挂载策略
- `enable.sh` / `disable.sh`：userscript 启停入口
- `scripts/patch_control_ui.py`：本机 OpenClaw Control UI patch 入口
- `CHANGELOG.md`：每次修复记录

## 7. 失败模式与回退

| 失败模式 | 行为 |
|----------|------|
| 找不到顶部刷新锚点 | 降级为右上角收起态图标，不阻塞聊天 |
| 发送按钮定位失败 | 显示本地错误状态；不影响核心插件 |
| 升级导致 userscript 失效 | 执行修复包更新 |
| Tampermonkey 已安装但不执行 | 切换 native fallback |
| 用户禁用脚本 | 回到手工命令模式 |

## 8. 安全与边界

- 胶囊不读取密钥，不写 `openclaw.json`
- 胶囊不拦截 OpenClaw 原生按钮行为
- 胶囊仅“发命令”，核心执行权仍在插件
- native fallback 只修补本机 Control UI 展示层，不改变 autosure 核心逻辑

## 9. 验收用例

1. 打开 chat 页，默认看到**顶部收起态图标 + 状态点**。
2. 收起态位于**刷新图标左边**。
3. 点击图标后，胶囊**向下展开**。
4. 第一排出现：`3` `6` `9` `∞`。
5. 第二排出现：`Stop` `Status`。
6. 点击 `3`，输入框自动发送 `/autosure 3`。
7. 点击 `Stop`，发送 `/autosure stop`。
8. 点击 `Status`，发送 `/autosure status`。
9. 点击外部区域，胶囊收起。
10. 若顶部锚点失效，胶囊降级为**右上角收起态图标**，而非右下角大浮层。
11. userscript 与 native fallback 在视觉结构与交互上保持一致。
12. 人工禁用胶囊后，手工 `/autosure` 仍完全可用。
