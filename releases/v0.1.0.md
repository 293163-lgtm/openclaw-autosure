# Release v0.1.0

**Autosure for OpenClaw** is a public release of a field-tested OpenClaw skill for safe unattended continuation.

It is designed for operators who want sessions to keep moving after benign interruption or between successful turns — **without collapsing into blind retries, message storms, or vague UI magic**.

## Highlights

- **Failure-aware auto-resume**
  - verified interruption paths
  - dedupe and cooldown
  - circuit breaker for repeated failures

- **Bounded `/autosure` continuation**
  - finite or unlimited continuation loops
  - human override remains first-class
  - `/autosure N` now starts the first frontstage continuation immediately, then continues the remaining rounds after each completed turn

- **UI control surface**
  - optional userscript UI capsule
  - native fallback patch path for local OpenClaw Control UI environments where userscripts do not execute reliably

- **Operator toolkit**
  - install scripts
  - repair/doctor tools
  - runtime validation helpers
  - deploy and handoff docs

## Product posture

This is **v0.1.0** because it is the first public release, not because it is toy-quality.

The implementation is already field-tested and engineered around real operator constraints:

- don’t retry blindly
- don’t hide stop semantics
- don’t trust one fragile surface
- keep the human in charge

## Notes

- Default product route remains the userscript UI capsule.
- On mini49, native fallback is the locally preferred working path because Tampermonkey on local Control UI has been unreliable.
- The packaged artifact for this release is included as `autosure.skill`.
